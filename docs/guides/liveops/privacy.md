# Privacy — Defaults That Don't Ship Broken

A solo dev cannot become a GDPR violator by accident. Defaults err on the side of "scrub it".

## Hard defaults

| Setting | Default | Why |
|---------|---------|-----|
| `send_default_pii` | `false` | Never send IP / username / email unless explicit opt-in |
| Device ID | hashed (sha256 + per-game salt) | No raw IDFA / Android ID |
| IP address | dropped at collector ingress | GDPR personal data |
| Username | omitted from envelope | Use opaque `player_hash` |
| Save data | never attached | Replay = inputs + snapshot, not save file |
| Email | never collected by engine | Game opts in if needed |
| Geo | country only, derived server-side | No city-level |
| Telemetry sample on minors | always 0% | Set when COPPA flag on |

`Nexus.toml`:

```toml
[diag.privacy]
mode                = "strict"      # strict | balanced | permissive
scrub_ip            = true
scrub_user_paths    = true          # /Users/alice/ → /Users/<user>/
scrub_env_vars      = ["HOME","USER","USERPROFILE","HOSTNAME"]
scrub_regex_file    = "scripts/diag/scrub.regex"
hash_device_id      = true
hash_salt           = "${PRIVACY_SALT}"   # rotate per game
retain_days_errors  = 90
retain_days_replay  = 30
retain_days_telemetry = 365
coppa_mode          = false
ccpa_do_not_sell    = true
```

## Scrub stages

```
emit → in-process scrub → adapter → collector ingress scrub → store
        (engine)                       (collector rule pack)
```

In-process scrub runs before the envelope leaves the player's machine. Collector scrub is defense-in-depth.

## Built-in regex pack

```
# scripts/diag/scrub.regex (default)
\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}\b   →  <email>
\b\d{3}-\d{2}-\d{4}\b                                  →  <ssn>
\b4[0-9]{12}(?:[0-9]{3})?\b                            →  <ccn>
/Users/[^/]+/                                          →  /Users/<user>/
C:\\Users\\[^\\]+\\                                    →  C:\Users\<user>\
/home/[^/]+/                                           →  /home/<user>/
```

Override per game in `scripts/diag/scrub.regex`. Custom patches add, never weaken.

## Opt-out wiring

In-game UI is mandatory. Generated by `nexus new` into `game/src/menu/privacy.rs`:

```
[ ] Send anonymous crash reports
[ ] Send anonymous gameplay analytics
[ ] Participate in A/B tests
[Manage data]  [Delete my data]
```

Settings persist in OS-appropriate location. Flipped off mid-session = stops within 1s, drops queued events.

## GDPR / CCPA / COPPA

| Right | Default behavior | Override |
|-------|------------------|----------|
| Access (GDPR Art. 15) | `nexus diag export --player-hash=$ID` returns all stored events | per game endpoint |
| Erasure (Art. 17) | `nexus diag forget --player-hash=$ID` purges errors+telemetry+replay | webhook to collector |
| Portability (Art. 20) | export is JSON | — |
| Do-Not-Sell (CCPA) | analytics off when flag on | — |
| Children (COPPA) | flag on → telemetry off, no analytics | per region detect |

## EU residency

| Region | Default collector | Override |
|--------|-------------------|----------|
| EU player | EU shard (Frankfurt) | `diag.collector_region=eu` |
| Other | nearest shard | per game |

`nexus new` ships docker-compose with two collector shards out of the box (EU + non-EU).

## Retention

Configured above. Hard-deleted by collector cron. Audit log retained 7 years.

## Audit

```bash
nexus diag privacy audit          # dumps current config + last scrub stats
nexus diag privacy test --input=samples/pii.txt   # runs regex pack offline
```

## Smoke test

```bash
nexus diag emit --message="alice@example.com crashed" --to=stdout
# expect: "<email> crashed" — not the raw email
```

## Rollback (loosen — discouraged)

```bash
nexus config set diag.privacy.mode permissive    # logs WARN every session
# in permissive mode, engine prints daily warning + UI banner
```
