# OTA / Delta Updates

Binary updates across platforms. Delta where possible. Always signed.

## Per-platform mechanism

| Platform | Channel | Delta? | Background? | Force? |
|----------|---------|--------|-------------|--------|
| macOS    | Sparkle | yes (bsdiff/xdelta) | yes | yes |
| Windows  | Squirrel.Windows / nexus-updater | yes (deltas via wyDelta) | yes | yes |
| Linux    | AppImage + zsync / Flatpak / Snap | yes | yes | depends |
| Android  | Google Play in-app updates | yes (Play Asset Delivery) | flexible/immediate | immediate |
| iOS      | App Store + background download | partial (app thinning) | yes (Background Tasks) | no (store policy) |
| Steam    | Steam pipeline | yes (bin diff) | yes | per build |
| itch.io  | butler push (diff patches) | yes | via app | per build |
| Web      | service-worker cache + content channel | yes | yes | reload |

## Engine abstraction

```toml
[ota]
provider = "auto"            # auto picks per platform
channel  = "stable"          # see live-content channels
check_interval_h = 6
delta = true
background = true
verify_signature = true
allow_downgrade = false
```

```bash
nexus ota check                    # poll
nexus ota apply --restart          # download + verify + restart
nexus ota rollback --to=<ver>      # opt-in if allow_downgrade
```

## Per-platform recipes

### macOS (Sparkle)

```xml
<!-- SUFeedURL in Info.plist -->
<key>SUFeedURL</key><string>https://ota.example.com/macos/appcast.xml</string>
```

```bash
# CI publishes appcast
nexus ota publish --target=macos \
  --appcast=ota/appcast.xml --bundle=MyGame-1.0.4.dmg \
  --ed25519-key=$KEY
```

### Windows (Squirrel-style, nexus-updater)

```bash
nexus ota publish --target=windows \
  --release=MyGame-1.0.4-full.nupkg \
  --delta-from=MyGame-1.0.3 \
  --feed=https://ota.example.com/windows/
```

### Linux (AppImage + zsync)

```bash
zsyncmake MyGame-1.0.4.AppImage
nexus ota publish --target=linux \
  --bundle=MyGame-1.0.4.AppImage \
  --zsync=MyGame-1.0.4.AppImage.zsync
```

### Android (Play in-app updates)

```kotlin
// AppUpdateManager API — engine wraps this
nexus.ota.androidCheck { update ->
  if (update.flexible) startFlexibleFlow()
  else if (update.immediate) startImmediateFlow()
}
```

`→ Google Play in-app updates docs` link below.

### iOS (background download + App Store)

App Store reviews block instant binary OTA. Use:

| Path | Use |
|------|-----|
| TestFlight | beta/canary builds, instant |
| App Store | stable, 24–48 h review |
| Background Tasks (`BGAppRefresh`) | warm caches, pre-pull content bundle |

Code patching is forbidden by App Review §3.3.2. **Engine code cannot OTA on iOS.** Scripts and assets via live-content can.

### Web (WASM)

```js
// service-worker.js auto-generated by nexus build --target=web
nexus.ota.web({ scope: '/', channel: 'stable' })
```

Reload after fetch; new WASM applied on next load.

## Delta strategy

| Strategy | When |
|----------|------|
| `bsdiff` / `xdelta` | small native binaries (< 100 MB) |
| `zsync` | AppImage / large monolithic files |
| `block-level` | platform default (Play, Steam) |
| `content-channel` | scripts/assets/shaders → live-content (no OTA needed) |

Aim: < 5% original size for most patches. If > 20%, ship full.

## Channels (mirror live-content)

| Channel | Audience | Auto-update |
|---------|----------|-------------|
| dev     | devs | manual |
| canary  | 1% | yes, immediate |
| beta    | opt-in | yes, flexible |
| stable  | everyone | yes, flexible |

`→ docs/guides/liveops/canary-and-rollback.md`

## Signing

| Platform | Trust root |
|----------|------------|
| Sparkle  | Ed25519 + EdDSA in appcast |
| Squirrel/nexus-updater | Code-sign certificate (Authenticode) + Ed25519 manifest |
| Android  | Play signing v3 |
| iOS      | Apple developer cert |
| Linux    | GPG + Ed25519 manifest |
| Steam    | Steamworks signature |

`→ docs/guides/release/codesigning/` (Agent 21)

## Rollback

| Platform | How |
|----------|-----|
| Sparkle/Squirrel | publish older release with higher build metadata + `allow_downgrade=true` |
| Android Play | halt rollout in console; users on prior version stay |
| iOS App Store | re-submit prior binary as new build |
| Steam | revert to prior branch |
| Web | flip live-content channel pointer |

Practically: prefer hotfix over rollback. Live-content rollback is seconds; OTA rollback is hours.
