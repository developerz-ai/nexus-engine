# Authoring — Localization (i18n)

> Fluent files per language under `locale/`. `nexus.mod.locale.t(key)` at runtime. Live language switch without restart.

## Format

[Fluent](https://projectfluent.org/) (Mozilla's localization system). Better than gettext for game text: supports plurals, conditions, branding tokens, parameter formatting natively.

`locale/en-US.ftl`:

```fluent
# Identifiers are lowercase-dashed.
mod-name = Healing Pack
mod-description = Adds craftable healing packs and a heal-over-time aura.

healing-applied = +{ $amount } HP
healing-applied-critical = Critical heal! +{ $amount } HP

# Plurals.
items-found = { $count ->
   [0] No items found
   [one] { $count } item found
  *[other] { $count } items found
}

# Brand tokens.
intro = Welcome to { -brand-name }!
-brand-name = My Cool Mod
```

## Manifest

```toml
[locale]
default = "en-US"
supported = ["en-US", "fr-FR", "de-DE", "ja-JP", "zh-Hans", "es-ES", "pt-BR"]
```

Add a language: drop `locale/<lang>.ftl`, list it in `supported`. No code change.

## Runtime

```rune
use nexus::mod::locale::{t, t_args, current_lang};

let msg = t("healing-applied");
let msg2 = t_args("healing-applied", #{ amount: 25 });
let lang = current_lang();   // e.g., "en-US"
```

Missing key → returns the key surrounded by `[]`: `[healing-applied]`. Logged at warn so authors notice.

Engine handles font fallback (CJK, RTL, etc.) automatically; no per-mod config needed for that.

## Live Switch

Player switches language in engine settings → engine emits `LanguageChanged { from, to }`. Mod's locale module re-loads files; cached strings invalidated. UI auto-rebuilds.

For mods that cache strings: subscribe to the event and recompute:

```rune
events.subscribe("engine.LanguageChanged", |_| {
    self.cached_strings = build_cache();
});
```

## Pluralization

Fluent's `select` expression handles every CLDR plural category. Engine ships the CLDR data; mod ships only its translations.

```fluent
killed-by = { $count ->
   [one] killed by one enemy
  *[other] killed by { $count } enemies
}
```

## Brand Tokens

Names that should never change across translations:

```fluent
-brand-mod = Healing Pack
-brand-author = sebi

credits = { -brand-mod } by { -brand-author }
```

## Date / Number Formatting

```fluent
last-saved = Last saved: { DATETIME($date) }
score = Score: { NUMBER($score, useGrouping: "true") }
```

Engine's Fluent runtime provides DATETIME/NUMBER/CURRENCY built-ins per CLDR data.

## RTL & Bidi

Languages: Arabic, Hebrew, etc. The engine's UI layout supports RTL. Mods using `nexus.mod.ui` get correct text direction automatically.

## Asset Localization

Per-language asset overlays:

```toml
[[overlays]]
target = "01HZ8XQK..."         # UI title screen
file = "overlays/title_screen_en.overlay.toml"
mode = "replace"
priority = 100
when_lang = "en-US"            # only applies when this language active

[[overlays]]
target = "01HZ8XQK..."
file = "overlays/title_screen_ja.overlay.toml"
mode = "replace"
priority = 100
when_lang = "ja-JP"
```

Engine swaps when language changes.

## Translation Workflow

`nexus mod locale add fr-FR` — generates `locale/fr-FR.ftl` from `locale/en-US.ftl` as the template with `# TODO:` markers on each key.

```fluent
# TODO: translate
mod-name = Healing Pack
```

`nexus mod locale missing fr-FR` — lists keys not yet translated.

`nexus mod locale check` — validates every supported language has every key; CI-friendly.

## Crowdsourcing

Common patterns:
- Host translations in the mod repo; PRs accept new languages.
- Use [Weblate](https://weblate.org/) or [Crowdin](https://crowdin.com/) and pull periodically.
- Engine's CLI exports/imports `.ftl` to these formats.

## Pitfalls

- Hard-coding English strings in `.rn` source: lint catches; CI rejects.
- Forgetting to add a new key to all `[locale].supported` languages: `nexus mod locale check` flags.
- Plural rules vary per language; never write `string + count + " items"`; use Fluent's `select`.
