# nexus-coder — Context Engineering

> How nexus-coder packs architecture, specs, and contracts into each subagent prompt without blowing context — and how the cache pays for it.

→ Models + cache awareness: `docs/specs/coder/models.md`
→ Tools that fetch context dynamically: `docs/specs/coder/tools.md`
→ Style reference for compressed config: `~/workspace/sebyx07/claude-code-bible/docs/11-compressed-config.md`

---

## Boundaries

- **Owns:** context pack assembly, cache strategy, eviction, retrieval heuristics.
- **Does NOT own:** the prompt content (subagent role file) · the live context window (the model owns it).

---

## The packing problem

Every subagent prompt costs tokens × every call × N parallel × every workflow. A naive pack of "all of `docs/`" = ~500k tokens, $0.50–$5 per call × thousands of calls = bankruptcy.

Constraint: keep per-subagent context ≤ 100k tokens, ≥ 70% of which is cache-hot.

---

## Layered context (front-to-back)

```
┌─────────────────────────────────────────────────────────┐
│ L0  System prompt (role rules, tool catalog)           │  ~2k tok   cached forever
├─────────────────────────────────────────────────────────┤
│ L1  Nexus constitution (vision + 12 principles)        │  ~3k tok   cached forever
├─────────────────────────────────────────────────────────┤
│ L2  Style guide (compressed-config Chapter 11)         │  ~1k tok   cached forever
├─────────────────────────────────────────────────────────┤
│ L3  Architecture map (system-map, tech-stack)          │  ~5k tok   cached forever
├─────────────────────────────────────────────────────────┤
│ L4  Relevant contract(s) — full text                   │  ~5k–15k   cached per task class
├─────────────────────────────────────────────────────────┤
│ L5  Primary spec(s) — full text                        │  ~5k–20k   cached per task
├─────────────────────────────────────────────────────────┤
│ L6  Referenced sibling specs (1 level deep)            │  ~5k–15k   cached per task
├─────────────────────────────────────────────────────────┤
│ L7  Repo map (paths + 1-line summaries, no contents)   │  ~3k–10k   cached per session
├─────────────────────────────────────────────────────────┤
│ L8  Hot code excerpts (only relevant fns)              │   var      not cached
├─────────────────────────────────────────────────────────┤
│ L9  Task instruction + outputs from upstream DAG nodes │   small    unique per call
└─────────────────────────────────────────────────────────┘
```

L0–L3 form the **stable prefix** — same bytes across thousands of calls. Anthropic / OpenAI prompt caching turns these from $5 to $0.50. → `https://docs.anthropic.com/en/docs/build-with-claude/prompt-caching`.

## How L4–L7 are picked

| Layer | Picker |
|---|---|
| L4 contracts | spec frontmatter `Depends on` table + grep for `docs/contracts/...` mentions in primary spec |
| L5 primary spec | task input (one or more spec paths from the workflow node) |
| L6 sibling specs | parse `→ docs/specs/...` cross-links in primary spec, follow 1 hop |
| L7 repo map | `crates/*/Cargo.toml` + first doc-comment of each `src/lib.rs` |

The picker is a deterministic pre-pass — no LLM call. Reproducible per task.

## Dynamic vs static context

```
static (pre-loaded into prompt)
    ┌──────────────────┐
    │ L0–L7 above      │   bounded, ≤ 80k tokens
    └──────────────────┘

dynamic (fetched on demand via tools)
    ┌──────────────────┐
    │ ReadFile         │
    │ ReadSpec         │
    │ GrepCodebase     │   pull only what the model asks for
    │ ListSpecs        │
    └──────────────────┘
```

Subagents are trained (via role prompt) to **start narrow and pull more via tools** rather than over-pack. Pattern reference: Aider's tree-sitter repo map. → `https://aider.chat/`.

## Repo map (L7) format

```
crates/
├── core/                   ECS, schedule, world, change detection
├── renderer/               wgpu backend, render graph
├── physics/                rapier wrapper, deterministic step
├── audio/                  cpal + spatial mixer
├── networking/             rollback netcode
├── agent-sdk/              JSON-RPC client, scenarios, replay
└── ...
games/
├── demo-fps/               integration test for renderer+physics+input
└── demo-rpg/               integration test for dialogue+inventory
docs/specs/
├── core/ {ecs,memory,jobs,hal,math,events}
├── renderer/ {overview,backend,pbr,shadows,gi,...}
└── ...
```

Generated by a deterministic script (`nexus coder context build-map`). Committed to repo so subagents see the same map across machines. Refreshed by CI on every merge to main.

## Cache strategy

| Provider | Mechanism | Used by nexus-coder |
|---|---|---|
| Anthropic | `cache_control` headers, 5-min TTL | yes, on L0–L3 (always), L4–L5 (session-scoped) |
| OpenAI | automatic prefix cache | yes, prefix order matches L0→L9 |
| Google Gemini | implicit prefix cache | yes |
| DeepSeek / others | varies | best-effort, fall back to no cache |

Target cache-hit rate ≥ 70% on the static prefix. Emit `cache.hit_ratio` in telemetry; alarm on < 50% drop. → `docs/specs/coder/telemetry.md`.

## Eviction & windowing

When a subagent exceeds 80% of its model's context window:

1. drop L8 hot-excerpts first (cheap to refetch via `ReadFile`).
2. summarize L6 sibling specs into 1-line bullets (architect-role subagent does this).
3. as last resort, summarize L5 primary spec — emit warning to telemetry.

Long tasks should be split into multiple DAG nodes rather than one giant context. The DAG **is** the long-context substitute.

---

## Cross-link discipline

Every spec includes `→ docs/...` link blocks (the Nexus convention). Picker uses these to traverse. If a spec doesn't link its contracts, picker emits `context.missing_link { spec, target_kind }` — fix with `nexus coder context lint`.

| Role | Extra context |
|---|---|
| `architect` | + `docs/architecture/05-adr/*.md`, + `docs/prior-art/*.md` |

| `triager` | minimal: L0–L3 + L7 only |

## Context build pipeline

```
task arrives
   │
   ▼
deterministic picker
   ├── load L0–L3 (cached, static)
   ├── load L4 contracts (from spec frontmatter)
   ├── load L5 primary spec
   ├── load L6 sibling specs (1 hop)
   ├── load L7 repo map (cached, session)
   ▼
assemble prompt (preserve order so cache lines up)
   │
   ▼
attach Vercel AI SDK cache headers per chunk
   │
   ▼
hand to subagent process
```

## Performance contract

| Metric | Target | Hard limit |
|---|---|---|
| Context build wall time | < 100 ms | < 500 ms |
| Static prefix token count | 5–15k | 25k |
| Total per-call prompt | < 50k | < 100k |
| Cache hit ratio (static prefix) | ≥ 70% | ≥ 50% (alarm) |
| Sibling-spec depth | 1 hop | 2 hops max |

---

## Error contract

| Code | Meaning | Caller action |
|---|---|---|
| `E_SPEC_MISSING_LINK` | spec lacks required cross-links | emit telemetry, run anyway |
| `E_CONTEXT_OVERFLOW` | even after eviction, prompt > window | split task, retry |
| `E_PICKER_NONDETERMINISTIC` | picker output varies across runs | bug; halt and report |
| `E_CACHE_HEADER_REJECTED` | provider rejected cache header | downgrade to uncached, warn |
