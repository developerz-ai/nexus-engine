# `scripts/` — Discovery

> An AI agent discovers every script in any Nexus repo by reading two files: `scripts/manifest.toml` (source of truth) and `scripts/index.json` (generated, agent-consumed).

## Two Files

| File | Format | Hand-edited? | Read by |
|---|---|---|---|
| `scripts/manifest.toml` | TOML | yes | humans, `index-scripts` |
| `scripts/index.json` | JSON | no — generated | AI agents, CI, doc gen |

`index.json` is regenerated by `scripts/index-scripts` from `manifest.toml` + each script's `--help --json` output. Pre-commit hook enforces freshness. CI fails if drift.

## `scripts/manifest.toml` Schema

```toml
schema_version = "1"

[meta]
generated_by   = "scripts/index-scripts"
last_generated = "2026-05-17T08:00:00Z"
repo_kind      = "engine"          # "engine" | "game"

[[script]]
name        = "build"
path        = "scripts/build"
lang        = "bash"               # bash | py | ts
category    = "build"              # bootstrap | build | test | deploy | liveops | dev | release | meta
description = "Compile workspace with sccache + features."
since       = "0.1.0"
idempotent  = true
flags = [
  { name = "profile",  type = "enum:dev|release|dist", default = "dev" },
  { name = "target",   type = "string", default = "host" },
  { name = "feature",  type = "string", multi = true },
  { name = "workspace", type = "switch" },
]
exit_codes = [
  { code = 0, meaning = "success" },
  { code = 4, meaning = "missing toolchain — run scripts/bootstrap" },
  { code = 5, meaning = "compile error" },
]
required_env = ["NEXUS_ENV"]
emits_artifacts = ["target/<triple>/<profile>/"]
test_file   = "scripts/build.bats"
```

## `scripts/index.json` Schema

```json
{
  "schema_version": "1",
  "generated_at": "2026-05-17T08:00:00Z",
  "repo_kind": "engine",
  "scripts": [
    {
      "name": "build",
      "path": "scripts/build",
      "lang": "bash",
      "category": "build",
      "description": "Compile workspace with sccache + features.",
      "since": "0.1.0",
      "idempotent": true,
      "flags": [
        { "name": "json", "type": "switch", "default": false, "source": "base" },
        { "name": "profile", "type": "enum", "values": ["dev","release","dist"], "default": "dev" }
      ],
      "exit_codes": [
        { "code": 0, "meaning": "success" }
      ],
      "required_env": ["NEXUS_ENV"],
      "help_snapshot_sha": "sha256:…"
    }
  ]
}
```

Base flags (`--help --json --quiet --verbose --dry-run --no-color --version`) are auto-merged into every entry — sources marked `"source": "base"`.

## Generation Algorithm (`scripts/index-scripts`)

```
1. read scripts/manifest.toml
2. for each [[script]] entry:
   a. assert path exists and is executable
   b. run `<path> --help --json` → must exit 0
   c. merge declared flags + base flags + parsed flags from --help-json
   d. compute sha256 of `<path> --help` plain output → help_snapshot_sha
   e. validate test_file exists
3. emit scripts/index.json (deterministic key order, 2-space indent)
4. emit scripts/index.md (human-readable table, for docs)
5. exit 0 if no drift vs. existing index.json, else 8 with --json errors[]
```

Pre-commit: `scripts/index-scripts --check`. CI: same, plus snapshot-diff.

## Agent Query Patterns

```bash
# list every script
jq '.scripts[] | .name' scripts/index.json

# list by category
jq '.scripts[] | select(.category=="deploy") | .name' scripts/index.json

# get flag list for a script
jq '.scripts[] | select(.name=="build") | .flags' scripts/index.json

# get exit-code meaning
jq '.scripts[] | select(.name=="build") | .exit_codes[] | select(.code==5)' scripts/index.json
```

nexus-coder subagent reads `scripts/index.json` once per session, caches it in context. `→ docs/specs/coder/tools.md`.

## Drift Detection

CI step `scripts/lint-scripts` runs:

```
scripts/index-scripts --check --json
```

Exits 8 on drift. Output:
```json
{ "ok": false, "errors": [
  { "code": 8, "message": "build flag --foo declared in --help but missing in manifest" }
]}
```

## Categories (closed enum)

| Category | Meaning |
|---|---|
| `bootstrap` | environment setup |
| `build` | compile / package |
| `test` | tests, scenarios, benches |
| `dev` | local dev loops |
| `deploy` | push to environment |
| `release` | tag, sign, publish |
| `liveops` | runtime ops on live game |
| `meta` | scripts about scripts (index, lint, new) |

New category = PR to this spec.
