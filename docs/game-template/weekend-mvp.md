# Weekend MVP — Friday 6pm to Sunday 8pm

> One solo developer. One AI agent fleet. 50 hours. A shipped game on Steam, itch, and the web. This is the design target the vision document commits to.

## The Setup

- **You**: a person with a game idea, an editor, a token budget, and a strong coffee maker.
- **Your team**: `gameplay-dev`, `balance-tuner`, `content-designer`, `qa-runner`, `perf-engineer` — the five agents shipped in `ai-agents/.claude/agents/` by `nexus new`.
- **Your engine**: Nexus, MIT, free, AI-first → `docs/initial/vision.md`.
- **Your goal**: a playable, polished, multiplayer-capable game. Not a tech demo. A game people install and play.

## The Premise For This Walkthrough

A 2D top-down co-op roguelike. Two players. Procedural dungeons. Pixel art. Steam Deck verified. Web playable. Source-available.

## Friday Evening — Foundation (6pm → midnight, 6h)

### 18:00 — `nexus new`

```bash
nexus new dungeonrats \
  --genre=roguelike \
  --style=pixel \
  --platforms=linux,windows,macos,web \
  --license=MIT \
  --git
cd dungeonrats
```

Output: 142 files in 4 seconds. Workspace compiles. `nexus run` opens a window with the genre's default scene (a procedural room, a controllable character). You have a running game.

### 18:05 — Read `Nexus.toml`. Edit two lines.

```bash
$EDITOR Nexus.toml
```

Set `[networking].model = "rollback"`, `[networking].max_players = 2`. Save. `nexus lint` confirms valid.

### 18:10 — Define the game in `specs/`

```bash
nexus generate scenario smoke
$EDITOR specs/features/core-loop.md
```

Write 200 words: what is the core loop, what is the win condition, what is the lose condition. This is the spec the agents will execute against all weekend.

### 18:30 — Spawn the first agent

```bash
nexus agent invoke gameplay-dev "implement the core loop from specs/features/core-loop.md"
```

Agent reads spec, runs `nexus generate entity Player`, `nexus generate system DungeonGen`, writes ~600 LOC + tests + a scenario. You watch the JSON event stream. 25 minutes later, `nexus test` is green.

### 19:00 — Procedural generation

```bash
nexus agent invoke gameplay-dev "implement procedural dungeon generation: 5 room types, BSP layout, doors, treasure rooms, boss room. Spec it first in specs/features/proc-gen.md."
```

Agent writes the spec, you approve via CLI, agent implements. 45 minutes.

### 19:45 — First playable

```bash
nexus run
```

You walk around a procedural dungeon. Bumping into walls. Doors open. Treasure spawns. It is a game.

### 20:00 — Dinner. Agent works in background.

```bash
nexus agent invoke content-designer "generate 30 enemy types with stats, behaviors, sprites. Use FLUX local for sprites. Spec each in specs/content/enemies/." --json &
```

Pixel art generated by FLUX, conditioned on `assets/style-refs/`. While you eat, 30 enemies land in `assets/generated/`.

### 21:00 — Combat

```bash
nexus agent invoke gameplay-dev "implement melee + ranged combat. Hitboxes, hurtboxes, knockback, i-frames. Frame data table in specs/balance/combat.md."
```

90 minutes. Combat works. `qa-runner` agent autospawns a scenario: `tests/scenarios/combat-smoke.toml`.

### 22:30 — Multiplayer

```bash
nexus agent invoke gameplay-dev "wire rollback netcode for 2-player co-op. Use the genre defaults. Spec the desync handling in specs/features/netcode.md."
```

The MOBA genre module covers most of it. 60 minutes. You open two clients on the same machine via `nexus run` × 2. Co-op works locally.

### 23:30 — Commit. Sleep.

```bash
git add -A && git commit -m "Friday: playable co-op roguelike core"
```

State at midnight: playable single-screen game, 30 enemies, combat, local co-op. ~3,500 LOC. All under spec. All tested.

## Saturday — Content & Polish (8am → midnight, 16h)

### 08:00 — Coffee. Run yesterday's tests.

```bash
nexus test --scenarios --json | jq '.failed'
```

Zero failures. Continue.

### 08:30 — World structure

```bash
nexus agent invoke content-designer "design 3 biomes: caves, sewers, library. Each: tileset, music, ambient sfx, enemy mix, boss. Spec each in specs/content/biomes/."
```

Spec runs first, you approve, agent generates: 3 tilesets (FLUX), 3 music tracks (Suno via `nexus add asset music`), 3 ambient beds, 3 boss specs. ~3 hours.

### 11:30 — Bosses

```bash
nexus agent invoke gameplay-dev "implement the three boss fights from specs/content/biomes/*/boss.md. Each is a state-machine boss with 3 phases."
```

Two hours. `qa-runner` writes a scenario per boss. All pass on first try for two of them; the library boss desyncs in multiplayer. `qa-runner` files an issue in `.nexus/issues/`.

### 13:30 — Lunch. Fix the desync.

```bash
nexus agent invoke gameplay-dev "fix issue .nexus/issues/2026-05-17-library-boss-desync.json. Use nexus agent replay bisect to find the divergent frame."
```

The bisect runs in ~90 seconds across thousands of frames. Agent finds it: a non-deterministic `HashMap` iteration. Patches with `BTreeMap`. Tests pass. 25 minutes including lunch.

### 14:00 — Meta progression

```bash
nexus agent invoke gameplay-dev "implement run-based meta progression: persistent currency, unlocks, characters. Spec in specs/features/meta.md."
```

90 minutes. Save system uses `nexus-engine`'s serialization layer. Save files are deterministic JSON; replays compose with saves.

### 15:30 — UI pass

```bash
nexus agent invoke content-designer "design the HUD, pause menu, run summary, and main menu. Pixel-art consistent with the rest. Spec in specs/content/ui.md."
```

Two hours. UI lives in `game/src/plugins/ui.rs`. egui-style declarative.

### 17:30 — Balance pass

```bash
nexus agent invoke balance-tuner "run 1000 simulated runs at headless speed 100x. Identify difficulty spikes. Adjust damage/HP curves in specs/balance/. Report a diff."
```

Headless sim runs at 100x; 1000 runs complete in ~15 minutes. Agent emits a balance diff. You approve. Re-runs. Curves smooth.

### 19:00 — Dinner. Agent on QA.

```bash
nexus agent scenario batch tests/scenarios/ --json > .nexus/qa-report.json &
```

Every scenario across both single and multiplayer. ~40 minutes. You eat. You come back. 2 failures. Both are flake-resistance issues (sleep-based assertions). Agent fixes both in 10 minutes.

### 20:00 — Web build

```bash
nexus build --target=web --profile=release
```

WASM, WebGPU. 8 MB compressed binary, 60 MB asset pack. Loads in 3 seconds on a fast connection. Co-op via WebRTC relay.

### 21:00 — Marketing site

```bash
nexus agent invoke content-designer "write the landing page copy + screenshots for web/apps/marketing. Three sections: pitch, gameplay GIFs, install."
```

Agent triggers `nexus run --record` to capture gameplay, ffmpeg-converts to GIFs, writes copy, deploys preview to localhost. 45 minutes.

### 21:45 — Steam page draft

```bash
nexus agent invoke content-designer "draft the Steam store page: short description, long description, tags, system requirements. Output as steamworks-import JSON."
```

You paste into Steamworks. Real screenshots from `nexus run --record`. 30 minutes.

### 22:15 — Bug bash

```bash
nexus run                       # play it for an hour
```

You play. You find 4 bugs. You file them via `nexus agent invoke qa-runner "log bug: [description]" `. Two are fixed before you finish playing.

### 23:30 — Commit. Sleep.

```bash
git add -A && git commit -m "Saturday: content, bosses, balance, web build"
```

State at midnight: 3 biomes, 3 bosses, 30 enemies, meta progression, web build, marketing draft. ~14,000 LOC. Tests green.

## Sunday — Ship It (9am → 8pm, 11h)

### 09:00 — Final balance + content review

```bash
nexus agent invoke balance-tuner "final 5000-run sim, two-player + solo. Generate distributions, flag any run with >40min duration."
```

Sim runs in 40 minutes at 100x. Three outlier runs flagged. Two adjustments. Re-sim. Clean.

### 10:30 — Performance pass

```bash
nexus agent invoke perf-engineer "profile a 30-minute run on minimum-spec hardware (Steam Deck profile). Identify any frame >16.6ms. Report fix candidates."
```

Headless sim runs the profile. Agent identifies: enemy AI pathfinding spike in the library biome. Patches with cached navmesh. 60 minutes.

### 11:30 — Localization

```bash
nexus agent invoke content-designer "extract all user-facing strings, translate to: es, fr, de, pt-br, ja, zh-cn, ru. Use the localization-sync tool."
```

LLM-translation, then a human-translator review hook (you opt to ship LLM-only for v0.1 and patch later). 45 minutes.

### 12:15 — Lunch. Build all platforms.

```bash
nexus build --target=all --profile=ship --package
```

~25 minutes parallel build across linux/windows/macos/web. Output: signed installers in `dist/`.

### 13:00 — Server deploy

```bash
nexus deploy server --env=prod
```

Fly.io, 3 regions (iad, fra, syd). Auto-scaling 2→16 instances. ~4 minutes including health checks.

### 13:30 — Web deploy

```bash
nexus deploy web --env=prod
```

Cloudflare Pages. Custom domain. CDN warm. ~90 seconds.

### 14:00 — Steam upload

```bash
nexus deploy steam --branch=playtest
```

Three depots uploaded. ~12 minutes. You set the branch live for a small invite list.

### 14:30 — itch.io upload

```bash
nexus deploy itch
```

Linux, Windows, macOS, web embed. ~5 minutes.

### 15:00 — Open the playtest

You DM 30 friends, 5 strangers from a Discord, post a Mastodon link. People play. You watch live telemetry:

```bash
nexus agent telemetry tail --filter='event=run_complete'
```

JSON events stream by. 14 concurrent players within an hour. Three crash reports auto-filed by the qa-runner agent from telemetry anomalies.

### 16:00 — Hotfix loop

For each crash:
```bash
nexus agent invoke gameplay-dev "reproduce and fix issue .nexus/issues/<id>.json"
nexus build --target=all --profile=ship --package
nexus deploy steam --branch=playtest
nexus deploy itch
nexus deploy web --env=prod
```

Three hotfixes in 90 minutes.

### 17:30 — Launch decision

Telemetry is clean for the last hour. No new crashes. p95 frame time under 12ms. Average session: 28 minutes. You decide to ship public.

### 17:45 — Promote to default branch

```bash
nexus deploy steam --branch=default
```

12 minutes. Steam page goes public the moment the build is on default and you flip "Coming soon" → "Available". Itch and web are already live.

### 18:00 — Tag the release

```bash
git tag v0.1.0
git push --tags
nexus agent invoke content-designer "draft the launch announcement: blog post, Mastodon thread, Reddit r/roguelikes post. Tone: humble, honest, link to source."
```

### 19:00 — Dinner.

### 20:00 — You hit the publish button.

Your game is live on Steam, itch, and the web. Source on GitHub, MIT. Server running in three regions. Cross-platform builds signed and distributed. 50 hours from `nexus new`. 

A real game. Not a demo.

## What Made This Possible

| Lever | Without Nexus | With Nexus |
|---|---|---|
| Engine setup | days | 4 seconds |
| Multiplayer | weeks | 60 minutes |
| Asset generation | weeks of art contracting | parallel to coding, hours |
| Cross-platform build | per-platform expertise | one command |
| Server deploy | DevOps engineer | one command |
| Bug bisect | hours of manual reproduction | 90 seconds via deterministic replay |
| Balance | gut feel | 5000-run sim in 40 minutes |
| Localization | translation agency | LLM + 45 minutes |

## What You Did Not Do

You did not: write boilerplate, configure a build system, debug a renderer, set up CI, write a netcode stack, design a save format, integrate Steamworks from scratch, write a content pipeline, build a web port from scratch.

You did: define the game, approve specs, play the game, decide when to ship.
