# Game Template — `scripts/` Overview

> What every freshly-`nexus new`'d game ships in its `scripts/` directory, and how it stays consistent across upgrades.

## The Critical Principle

AI agents work best with stable, parameterized, tested CLIs. The `scripts/` convention is identical between the engine repo and every game built from `nexus new`. A dev (with their AI) walks into any Nexus project and finds the same commands in the same place — `scripts/build`, `scripts/test`, `scripts/deploy` — same flags, same exit codes, same `--json` envelope.

## What `nexus new` Seeds

| Bucket | Path | Mutability |
|---|---|---|
| seeded entry scripts | `scripts/<name>` | overwritten on `nexus upgrade` |
| seeded shared utils | `scripts/lib/<name>.<ext>` | overwritten on `nexus upgrade` |
| seeded tests | `scripts/<name>.{bats,test.py,test.ts}` | overwritten on `nexus upgrade` |
| manifest | `scripts/manifest.toml` | merged on `nexus upgrade` (your custom entries preserved) |
| generated index | `scripts/index.json` | regenerated by `scripts/index-scripts` |
| your scripts | `scripts/custom/<name>` | NEVER touched by upgrades |
| your tests | `scripts/custom/<name>.{bats,test.py,test.ts}` | NEVER touched |

## Seeded Set Summary

20+ scripts covering dev loop, build, test, scenario/replay, asset gen, secrets, deploy/release, liveops (crash + canary + flags), AI orchestration. Full table: `→ docs/game-template/scripts/seed-list.md`.

## Contract Inheritance

Every seeded game-repo script conforms to the same contract as the engine repo:

| Required | Reference |
|---|---|
| `--help` exits 0, non-empty | `→ docs/specs/scripts/cli-contract.md` |
| `--json` produces parseable envelope | same |
| `--dry-run` short-circuits side effects | same |
| `--env` required for env-touching scripts | same |
| sibling test asserting the 4 mandatory cases | `→ docs/specs/scripts/testing.md` |

Custom scripts MUST follow the same rules. Enforced by `scripts/lint-scripts` in CI.

## Extension

| Want to | Do |
|---|---|
| add a new script | `scripts/new-script --name foo --lang py --category liveops` — lands in `scripts/custom/` |
| modify a seeded script's behavior | edit it; understand `nexus upgrade` will overwrite — better: shadow it under `scripts/custom/foo-extended` |
| remove a seeded script | remove from `scripts/manifest.toml`; CI lint will flag the orphan file |
| change a seeded script's flag set | open PR upstream to the template |

Rules: `→ docs/game-template/scripts/extension-rules.md`.

## Upgrade Flow

```bash
nexus upgrade --json --dry-run
```

Shows planned changes:
- which `scripts/*` will be overwritten
- which `scripts/lib/*` will be overwritten
- which `scripts/manifest.toml` entries will be merged
- `scripts/custom/*` always preserved

Run without `--dry-run` to apply. `scripts/index-scripts` regenerates the index automatically.

## Why the Same Shape Across Repos

| Without convention | With convention |
|---|---|
| every game invents its own commands | one set, learned once |
| AI agents must read each repo's README | AI agents read `scripts/index.json` |
| CI configs diverge | CI calls the same scripts everywhere |
| onboarding cost: hours | onboarding cost: `scripts/onboard` |
